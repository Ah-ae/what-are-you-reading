최근 커밋 5개의 메시지 스타일을 확인하고, 현재 변경 사항에 대한 커밋을 생성해줘.

## 규칙

1. `git log --oneline -5`로 최근 커밋 메시지 양식을 확인
2. `git diff --staged`와 `git diff`로 변경 사항을 파악
3. 최근 커밋과 동일한 양식(예: `type: 설명`)으로 커밋 메시지를 작성
4. 변경된 파일들을 `git add`하고 커밋을 생성

## 인자가 있는 경우

$ARGUMENTS 가 비어있지 않으면, 전체 변경 사항이 아닌 해당 인자에서 언급한 내용과 관련된 변경 사항만 골라서 커밋해줘.
나머지 변경 사항은 staging하지 않고 남겨둬.

## 커밋 메시지 타입

- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `update`: 기존 기능 개선/변경
- `refactor`: 리팩토링
- `chore`: 설정, 빌드 등 기타 변경
